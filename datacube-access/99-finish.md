
We have learned in this tutorial how to deploy and use the EOEPCA Datacube Access Building Block.

### Key Takeaways

- **Datacube metadata is essential**: The `cube:dimensions` and `cube:variables` fields tell processing tools how to interpret data as multi-dimensional arrays
- **Datacube Access filters collections**: Only properly-structured datacube collections are exposed
- **Standards-based**: Built on STAC, OGC APIs, and the emerging GeoDataCube API
- **Integration-ready**: Designed to work with openEO and other processing tools


### Next Steps
